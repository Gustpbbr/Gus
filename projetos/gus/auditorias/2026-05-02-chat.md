# Auditoria — porta Claude Chat (02/05/2026)

Auditoria multi-especialista de toda a superfície que envolve a porta Claude
Chat: bootstrap, estado dinâmico, MCP server, Drive sync, curador, ingest
pipeline, demandas. Encontrou 1 bug crítico ativo (curador 100% errando há
3 dias) + ~30 itens menores de hardening, arquitetura e documentação.

## Itens implementados (commit `e9d4046` + sessão atual)

1. **OPS-1 / curador KeyError** — `format()` → `replace()` em
   `_render_prompt`. Bug crítico que zerava ingestão há 3 dias.
2. **SEC-2 / fail-closed** — `_create_app()` com `AuthMiddleware(expected_header=None)`
   quando AUTH_DISABLED && sem URL_SECRET. Retorna 503 em tudo (exceto /health).
3. **SEC-3 / whitelist user_id** — `ALLOWED_USER_IDS = {"gustavo", "gus"}`
   em `ingestar_fragmento`.
4. **QUAL-1 / clamp confianca** — `_validar_fragmento` aplica `max(0, min(1, c))`.
5. **QUAL-2 / valida tipo** — TIPOS_VALIDOS, CAMADAS_VALIDAS, AREAS_VALIDAS
   contra enum gus-18. Inválido → default + log warning.
6. **OPS-3 / retry git push** — 4 tentativas com backoff 2/4/8/16s no
   `ingest_mem0_from_chat.py`. `-u origin main` explícito.
7. **OPS-7 / processados-erro** — falhas no curador agora movem pra
   `inbox-mem0-from-chat/processados-erro/AAAA-MM/<file>` com header de motivo.
8. **OPS-5+8 / headers e vars** — logs/comentários atualizados pra
   "Anthropic+OpenAI". Vars renomeadas pra `MODEL_CURADOR_ANTHROPIC` /
   `_OPENAI`. Vars antigas mantidas como fallback.
9. **ARCH-3 / bootstrap MCP > Drive** — aviso explícito sobre Drive stale
   + recomendação `read_repo_file`.
10. **ARCH-5 / brain gus no estado** — `gerar_estado_atual_chat.py`
    inclui bloco "Auto-observações do Gus" com últimos 8 fragmentos.
11. **DOC-1 / _estado-atual.md** — atualizado pra 02/05.
12. **DOC-2 / gus-28-passo2** — Etapa 1 vira URL_SECRET, Bearer migra
    pra anexo no fim. Etapa 3 explica fail-closed.

## Como verificar os fixes em produção

Após Gustavo setar `MCP_URL_SECRET` e Railway redeployar:

```bash
# 1. Health (público)
curl https://gus-mcp-server-production.up.railway.app/health
# → {"status": "ok", "service": "gus-hub-mcp"}

# 2. Path antigo deve dar 404
curl https://gus-mcp-server-production.up.railway.app/mcp
# → 404

# 3. Ingest fragmento via MCP (de dentro do Chat) com user_id inválido
# → {"ok": false, "error": "user_id inválido: 'foo'. Aceitos: ['gus', 'gustavo']"}

# 4. Validar curador (após próximo cron 30min do ingest-mem0-from-chat):
# Verificar _log/resumos-mem0/2026-05-02.md → não deve ter mais "status: erro"
# com KeyError 'conteudo'. Status esperados: "salvo" ou "descartado".
```

Para o Telegram (curador a cada 3 turnos):
- Bot rodando no Railway puxa fix de `hub/curador.py` quando merge for pra
  main + Railway redeploy.
- Próxima conversa de 3+ turnos no Telegram: log `_log/resumos-mem0/AAAA-MM-DD.md`
  deve ter entrada com `status: salvo (curador: haiku, ...)` em vez de
  `status: erro`.
